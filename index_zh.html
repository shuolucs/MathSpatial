主要发现

MathSpatial 揭示了当前 MLLM 在空间推理上的根本性局限。

巨大的人机差距

人类准确率 96.3%，最强 MLLM（GPT-5）仅 58.5%，空间推理上存在约 37.8 个百分点的差距。

抽象演绎最难

多数模型在几何量计算（GPC）上低于 5%；该子类最抽象，需要多步数值推理。

单纯扩规模无效

Qwen2.5-VL-72B（19.7%）相较 7B 版本（17.8%）几乎无提升，说明空间推理无法仅靠扩模解决。

在 MathSpatial 上训练有效

微调模型在结构化推理轨迹监督下持续提升准确率，并将 Token 用量降低约 20%–30%。

错误分析

细粒度错误分类揭示了不同模型族的失败模式差异，凸显空间推理中的根本难点。

（a）6 个子类上基线的错误频率分布。（b）按失败类型汇总的总体错误率。

推理断层（34.4%）

最主要的失败模式：模型推理链不完整或不一致。在 MathSpatial-Corpus 上微调可降低该类错误。

几何约束违背（33.0%）

输出违背正交投影或可见性约束。Claude 系列尤其容易出现该类错误。

投影误解（12.6%）

模型误读俯视图、侧视图或正视图。Gemini-2.5-flash 在该类错误上尤为突出。

特征 / 尺度 / 演绎

特征错误（10.5%）：遗漏部件。尺度错误（7.2%）：未保持大小关系。演绎失败（2.3%）：无法综合多条线索。

要点：保证底层几何一致性与维持连贯多步推理这两大核心挑战，当前 MLLM 仍未很好解决。细粒度错误分类为针对性改进提供了诊断基础。

原子级标注

MathSpatial-Corpus 中每道题均配有结构化推理轨迹，将空间推理拆解为三种原子操作：关联（Correlate）、约束（Constrain）与推断（Infer）。

C

关联 Correlate

建立几何特征在跨视图间的对应关系。

K

约束 Constrain

套用几何规则、投影约束与一致性条件。

I

推断 Infer

由证据推导隐藏属性、度量或最终答案。

（a）原子操作占比：Infer ~42% > Correlate ~33% > Constrain ~25%。 （b）平均轨迹长度：5.1 步。

轨迹 JSON 示例

{ "reasoning": [ { "op": "Correlate", "step": "The front and left views are identical isosceles triangles." }, { "op": "Constrain", "step": "The top view is a circle, constraining the base to be circular." }, { "op": "Infer", "step": "A solid with triangular projections and circular top is a cone." } ], "final_answer": "C" }

消融 — 移除单一原子操作的影响

消融：保留三种原子操作的完整解答，与依次移除 Correlate / Constrain / Infer 后退化输出对比；移除任一类都会导致不同失败模式。

子类定义 — 11 种细粒度任务

整体识别（4 种子类）

  • C3M — 复杂三视图与立体匹配
  • 3VM — 立体模型与三视图匹配
  • IVI — 错误视图辨识
  • CR — 部件识别

生成式推理（3 种子类）

  • MVC — 缺失视图补全
  • VT — 视点变换
  • PRA — 投影规则应用

抽象演绎（4 种子类）

  • GPC — 几何量计算
  • FPBI — 功能与物理行为推断
  • SCP — 结构变化预测
  • PPCD — 路径规划与碰撞检测
数据构建流程 — 多阶段质检
35,428 条原始候选 → 人工筛选（61.1%） → 去重（MD5 + SBERT + GPT-4.1 视觉） → 几何一致性检查 → 10,000 道最终题目

来源：百度文库、组卷网、省级试题库等。过滤阈值：≥0.90（剔除）、0.85–0.90（人工复核）、<0.85（保留）。约 1K 条进入人机协同复核，60% 修正后保留、40% 删除。

人工评测协议

被试：80 名初高中生 · 冗余：每题 20 人次 · 总作答：40,000

条件：闭卷、禁用计算器与软件，仅纸笔作答

结果：微平均准确率 = 96.3%
